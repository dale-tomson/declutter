# v1.1.2 - Bug Fixes

**Release Date:** 2025-12-07

## Overview

This patch release fixes UI state management issues and improves the CI/CD workflow reliability.

## Fixed

### Folder Selection After Organization
- **Fixed folder selection state** after organizing files
  - Users can now immediately select a new folder after organization completes
  - Organize button is properly disabled until a new folder is selected
  - Clear visual feedback with updated label: "No folder selected - Select a folder to organize more files"
  - Improves workflow clarity and prevents confusion

### CI/CD Workflow Improvements
- **Fixed Windows resource file handling** in GitHub Actions
  - Added conditional file existence checks before moving `.syso` files
  - Added fallback handling for different file naming patterns
  - Added debug output to verify resource file generation
  - Prevents `mv: cannot stat` errors during automated builds

## Technical Changes

### Modified Files
- `internal/ui/ui.go` - Reset folder selection state after organization
- `.github/workflows/release.yml` - Improved file handling robustness

## Impact

✅ **Better UX** - Clear workflow completion and ability to organize multiple folders in one session  
✅ **Reliable CI/CD** - Windows builds no longer fail due to file handling issues  
✅ **Improved Clarity** - Users understand when they need to select a new folder
